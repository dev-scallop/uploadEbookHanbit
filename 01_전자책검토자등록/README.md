# 전자책 검토자 자동 등록 시스템

## 📋 개요
Google Play Console에서 전자책 검토자를 자동으로 등록하는 Python 기반 자동화 시스템입니다.

## 🎯 주요 기능
- 엑셀 파일(.xlsx, .csv) 기반 일괄 검토자 등록
- 직관적인 GUI 인터페이스
- 실시간 진행률 표시 및 로그 출력
- 등록 결과 자동 저장 및 리포트 생성
- Chrome 세션 유지를 통한 효율적인 작업 처리

## 🛠️ 시스템 요구사항
- Python 3.7 이상
- Google Chrome 브라우저
- Google Play Console 접근 권한이 있는 계정

## 📦 설치 및 설정

### 1. 필요한 패키지 설치
```bash
pip install pandas openpyxl selenium webdriver-manager requests beautifulsoup4
```

### 2. 엑셀 파일 준비
다음 3개 컬럼을 포함한 엑셀 파일을 준비하세요:
- `이름`: 검토자 이름
- `도서명`: 등록할 전자책 제목
- `지메일`: 검토자 Gmail 주소

#### 샘플 데이터 형식:
| 이름 | 도서명 | 지메일 |
|------|--------|--------|
| 김철수 | Python 프로그래밍 기초 | test1@gmail.com |
| 이영희 | 데이터 분석 입문서 | test2@gmail.com |

## 🚀 사용 방법

### 1. 프로그램 실행
```bash
python "[개발중] 전자책자동등록.py"
```

### 2. GUI 사용법
1. **파일 선택**: "파일 선택" 버튼을 클릭하여 엑셀 파일을 선택
2. **데이터 확인**: 선택한 파일의 데이터가 미리보기 테이블에 표시됨
3. **등록 시작**: "등록 시작" 버튼 클릭
4. **로그인**: Chrome 브라우저가 열리면 Google 계정으로 수동 로그인
5. **자동 처리**: 시스템이 각 도서에 대해 검토자를 자동 등록
6. **결과 확인**: 등록 완료 후 "결과 저장" 버튼으로 결과 엑셀 파일 생성

## 📊 주요 화면 구성

### GUI 섹션
1. **파일 선택**: 엑셀 파일 업로드 및 경로 표시
2. **데이터 미리보기**: 업로드된 데이터의 테이블 형태 표시
3. **등록 실행**: 시작/중지 버튼 및 진행률 표시
4. **실행 로그**: 실시간 작업 로그 출력

## 🔧 설정 및 커스터마이징

### 로그 설정
- 로그 파일은 `logs/` 디렉토리에 자동 저장
- 파일명 형식: `ebook_registration_YYYYMMDD_HHMMSS.log`

### Chrome 설정
- 사용자 데이터는 `chrome_user_data/` 디렉토리에 저장
- 세션 유지를 통해 반복 로그인 불필요

## 📈 결과 분석

### 등록 결과 파일
등록 완료 후 생성되는 엑셀 파일에는 다음 정보가 포함됩니다:
- `name`: 검토자 이름
- `book_title`: 도서명
- `email`: 검토자 이메일
- `status`: 등록 상태 (SUCCESS/FAILED)
- `error`: 실패 시 오류 메시지

## ⚠️ 주의사항

### 보안 관련
- Google 계정 정보는 브라우저 세션에서만 관리
- 2단계 인증(2FA) 설정된 계정의 경우 수동 로그인 필요
- 세션 정보는 로컬에만 저장되며 외부 전송 없음

### 사용 제한
- Google Play Console의 정책 및 제한사항 준수
- 과도한 요청으로 인한 일시적 차단 가능성
- 각 요청 간 2초 대기 시간 설정

- Chrome 브라우저 최신 버전 유지
