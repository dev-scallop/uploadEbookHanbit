# 🎯 프롬프트 업데이트 (v2.2) - 브랜드 마케팅 전문가 관점

## 📝 업데이트 날짜
2025년 1월 6일

## 🎨 주요 변경사항

### 1. 역할 정의 명확화
```
기존: "아래 신간안내서 원문에서 도서 정보를 추출하고..."
개선: "너는 브랜드 마케팅 전문가다."
```
→ AI가 마케팅 전문가 페르소나를 명확히 인식

### 2. 브랜드 마케팅 전략 추가

#### ✅ 타깃 독자 맞춤화
- 기획서 분석 → 대상 독자 파악 (교사, 학생, 일반인, 전문가)
- 독자 유형별 언어와 메시지 차별화

#### ✅ USP (Unique Selling Proposition) 강조
- 이 책만의 차별화된 강점 부각
- 경쟁 도서 대비 우위 요소 명확화

#### ✅ 광고 카피 작성 구조
```
1. 짧은 헤드라인 (임팩트)
2. 2-3줄 설명 (흥미 유발)
3. 핵심 특징 리스트 (구체적 가치)
```

## 📋 세부 가이드라인

### promo_line (헤드라인)
**목적**: 3초 안에 시선 사로잡기

**작성법**:
- 파워 워드: "혁신", "필수", "완벽한", "최고의"
- 구체적 숫자: "단 7일", "500개 예제", "1위"
- 강렬한 문장형: 의문문 또는 단정문

**예시**:
```
✅ "STEM 교육의 새로운 기준"
✅ "7일 만에 정복하는 파이썬"
✅ "교사들이 가장 많이 선택한 교재"

❌ "좋은 STEM 교육 책"
❌ "파이썬을 배울 수 있는 책"
```

### intro_summary (2-3줄 설명)
**목적**: 흥미 유발 + 계속 읽게 만들기

**3단계 구조**:
```
1문장: 독자 고민/니즈 언급
  예) "경제학이 어렵게만 느껴지셨나요?"

2문장: 책의 솔루션 + 차별점
  예) "이 책은 실제 뉴스와 사례로 복잡한 개념을 쉽게 풀어냅니다."

3문장: 기대 효과/변화
  예) "읽고 나면 경제 뉴스가 저절로 이해됩니다."
```

### recommendation (대상 독자별 메시지)
**목적**: "이 책은 바로 나를 위한 것"이라는 확신

**패턴**:
```
- "~하는 분" + 구체적 상황 묘사
- "~을 원하는 분" + 제공 가치
- "~에 고민하는 분" + 해결책
```

**예시**:
```
초보자용:
✅ "처음 배우는 분도 따라하기 쉽게 단계별로 구성했습니다"
❌ "초보자에게 적합합니다"

실무자용:
✅ "현장에서 바로 적용 가능한 200개 실전 예제를 담았습니다"
❌ "실무에 도움이 됩니다"

교육자용:
✅ "학생들이 흥미를 갖고 참여하는 수업 자료가 필요하다면 최적입니다"
❌ "교사에게 유용합니다"
```

### feature (특징 리스트)
**title (10자 이내)**:
- 명사형 키워드
- 카테고리화된 제목

**desc (50-80자)**:
- 실질적 혜택 중심
- 구체적 결과 제시

**예시**:
```
title: "실무 예제"
desc: "업계 전문가가 검증한 200개 예제로 실무 감각을 키울 수 있습니다"

title: "동영상 강의"
desc: "QR코드로 바로 연결되는 50개 무료 강의로 이해를 돕습니다"
```

### publisher_review (출판사 관점)
**요소**:
1. 기획 의도
2. 시장 분석
3. 타깃 독자
4. 차별점

**예시**:
```
✅ "현장 교사들의 목소리를 담아 기획했습니다. 이론과 실습이 균형잡힌 구성으로 대학 교재는 물론 독학용으로도 완벽합니다."

❌ "많은 분들에게 도움이 되는 책입니다."
```

## ✅ DO / ❌ DON'T

### ✅ 해야 할 것

1. **구체적 숫자 활용**
   ```
   ✅ "200개 예제", "50개 동영상", "7일 과정"
   ❌ "많은 예제", "풍부한 자료", "짧은 기간"
   ```

2. **감정과 니즈 공감**
   ```
   ✅ "어렵게만 느껴지셨나요?"
   ✅ "실무에서 바로 써먹고 싶으신가요?"
   ❌ "이 책은 좋습니다"
   ```

3. **능동적 문장**
   ```
   ✅ "~할 수 있습니다", "~를 경험하세요"
   ❌ "~하게 됩니다", "~이 가능합니다"
   ```

4. **짧고 강렬한 표현**
   ```
   ✅ "단 7일. 파이썬 마스터."
   ❌ "7일이라는 짧은 시간 동안 파이썬을 완벽하게 배울 수 있습니다"
   ```

5. **차별화 포인트 부각**
   ```
   ✅ "국내 유일 STEM 통합 교재"
   ❌ "STEM 교재"
   ```

### ❌ 하지 말아야 할 것

1. **과도한 수식어**
   ```
   ❌ "매우 좋은", "정말 유익한", "엄청 재미있는"
   ✅ "검증된", "실전형", "몰입형"
   ```

2. **추상적 표현**
   ```
   ❌ "좋은 책", "유익한 내용", "알찬 구성"
   ✅ "200개 예제", "단계별 학습", "프로젝트 중심"
   ```

3. **긴 문장**
   ```
   ❌ "이 책은 초보자부터 전문가까지 누구나 쉽게 이해할 수 있도록 구성되어 있으며..."
   ✅ "초보부터 전문가까지. 모두를 위한 완벽한 구성."
   ```

4. **전문 용어 과다**
   ```
   ❌ "OOP 패러다임 기반 객체지향 설계 방법론"
   ✅ "객체지향 프로그래밍 핵심 원리"
   ```

5. **평범한 문구**
   ```
   ❌ "학습에 도움이 됩니다"
   ✅ "실력이 달라집니다"
   ```

## 📊 작성 체크리스트

- [ ] 파워 워드 또는 숫자 포함

## 💡 팁

### 타깃 독자 파악하는 법
```
기획서에서 찾아야 할 키워드:
- "교사용", "학생용", "일반인 대상"
- "초급", "중급", "고급"
```

```
비교 포인트:
1. 콘텐츠: 다른 책에 없는 내용?

3. 분량: 더 많거나 집약적?
4. 저자: 특별한 경력?
5. 부가자료: 동영상, 코드, 템플릿?
```
